3月23日10-11点，和成都前端团队开会，讨论前端技术发展方向。

+ 从代码审核讨论技术上大家的一些共性问题：不写注释，不规范的命名，不规范的代码风格等。

## 和各个成都前端团队成员单独聊天

主要沟通思路：

+ 强调了成都注重生活与工作的平衡，并尽可能了解员工的生活和难处，帮助员工解决生活上的问题，鼓励员工坚持做一些有意义的事情，而不是在休息时只会刷手机。
+ 了解了员工的诉求和想法，并尽可能安排理想的工作，同时咨询了员工对现有工作安排的不合理之处以及如何解决。
+ 尽可能引导员工往更广、更细致的方向发展，而不是单纯为了完成任务。
+ 和员工制定一些未来的目标，以及发展方向。

## 问题与总结

在本次出差中，我尽可能地帮助成都前端团队解决了一些问题，特别是前端开发工程师在职业发展上的局限性问题，拓宽了部分员工的视野，让他们意识到自己可以做的事情不仅仅是写代码，而是可以做更多的事情。在很多方面都存在着可以深耕的机会，只需要员工们自己去发现、去探索和尝试。

还存在一些问题，如成都的一些同事比较喜欢薅羊毛，可能存在一定的潜在风险，已经和人力资源部和张海新沟通，并将在后续多关注团队情况。此外，成都的食堂也存在一些问题，需要进一步改进。

总之，此次出差使我更深入地了解了成都前端团队的情况，为团队的发展提供了一些有益的建议和帮助。
