# cursor 使用体验

## 背景

> 我要开始更新技术文献了~，近几个月写的有点太偏了，还是得回归正轨，主业还是程序员。

从2022年开始，我个人使用 github copilot 的 ai 开发助手已经2年多了，而且是主力开发工具，也简短写过一些使用评价。大概今年7月份的时候，我仅仅使用了几天的 cursor ，我就决定使用 cursor 来替代 github copilot 作为主力开发工具。并开始进行包月了。

市面上比较多的介绍还是说 从 0-1 可以用 AI 来开发一个项目，但我这里想说的是，从 1-100 的角度来说明，这个工具和 github copilot 相比，的优越性，以及我实际使用的一些 prompt。

> 如果你之前用的是 vscode， 那 cursor 的编辑器初始化的时候可以让你和 vscode 完美迁移。几乎就是 升级版的 vscode。集成了所有 vscode 的配置与插件。

## AI 对话能力

cursor 和 copilot 其实都有，不过我 cursor 用下来更强大，更智能。他可以直接在编辑器中，直接和你对话，并且给出代码的补全。更有意思的是，他的体验更 好，可以一键 Apply（更新带代码中）。

这里最有名的就是一个 Cloudflare 的工程师 rickyrobinett 分享她8岁女儿用 cursor 写了一个 app 的twitter 并获得了10w点赞。b 站有搬运视频：<https://www.bilibili.com/video/BV1mrWYe4EWa/>；全程没有写一行代码。都是提示词完成代码编辑。

## 强大的代码助手

其实我个人认为 AI 暂时并不能取代人类，因为它并没有特别强大的创造力，更多的是用于工具来提效。 以前很多重复性的工作，在 cursor 中，你可以更加高效的完成。其中它体验最好的就是 它的 tab 键补全。

它的代码补全，并不针对单行，而是根据上下文，比如 你引入了某个库，那它可以根据你当前的代码，在文件最上面进行 import 补全。

当然补全 import 只是基操，还有 ts 类型补全，比如某个库的 ts 类型，你需要去查，但它可以做到直接补全。

比如你有一个代码，里面有10个变量，而你想给他们都加上注释，那你只需要写一行注释，cursor 就会自动的给其他变量加上注释。你要做的只是微调而已，同理，方法注释也是。

## prompt 实战案例

接下来我将通过几个具体的例子来展示 Cursor 的强大功能。

第一个例子展示的是 TypeScript 类型补全能力。在使用 TypeScript 重构代码时，经常会遇到类型报错的情况。此时只需右键点击错误提示，Cursor 就能智能分析并给出解决方案。比如在这个例子中，Cursor 识别出错误是由于 defineModel 缺少初始值导致的，并给出了正确的使用方式。这让我学到了 defineModel 的新用法，无需查阅 Vue 官方文档就能快速解决问题。

这种智能诊断和解决方案不仅限于类型错误，对于代码中出现的任何报错，都可以通过右键让 Cursor 来协助解决。这大大提升了开发效率和学习体验。

![ts 类型补全](/imgs/cursor/ts类型补全.gif)

接着看下第二个例子也是 ts，但我想说明的是 cursor 的上下文理解能力。当我定义了 一个 interface 之后，它会自动生成 相关内容（这个是基本操作）。但接着它会提示你，哪些上下文要改，而我需要做的，就是按几下 tab 键，就完成了这个文件的ts重构。这个体验真的超赞。

![上下文理解能力](/imgs/cursor/上下文理解能力.gif)

下面这个例子是跨文件理解能力，我刚建了一个文件想写本文的文档（talk-about-cursor.md），cursor 就帮我在路由中联想到了，甚至帮我补全了代码。这点在 github copilot 中是没有的。我在实战中也发现了不少可以跨文件联想的场景，非常实用。

![跨文件理解能力](/imgs/cursor/跨文件理解能力.gif)

当然还有很多，我就不一一展示了，不停按 tab 就完成开发的感觉真的太好了。
