# 【翻译】Cursor 的 12 个技巧

如何有效地使用Cursor AI IDE 的十二个关键方法 - 来自 Cursor 设计负责人 ryolu_

1. 设置明确的项目规则：在开始前设置5-10条清晰的规则，让 Cursor 了解项目结构和限制条件。可以使用 "/generate rules" 命令为现有代码库生成规则。

2. 编写具体的提示：详细说明技术栈、期望行为和限制条件，就像编写简洁的规格说明。

3. 逐文件工作：按小块生成、测试和审查代码，保持专注。

4. 先编写测试：先编写测试，锁定它们，然后生成代码直到所有测试通过。

5. 审查 AI 输出：始终检查 AI 生成的代码，修复任何问题，并告诉 Cursor 将修复后的代码作为示例。

6. 使用范围限定：使用 @ file、@ folders、@ git 等命令将 Cursor 的注意力限定在代码库的相关部分。

7. 保存设计文档：将设计文档和检查清单保存在 .cursor/ 目录中，使 AI Agent 能够获取完整上下文。

8. 遇到错误直接修改：如果代码有错，直接自己修改比解释更有效，Cursor 从编辑中学习更快。

9. 利用聊天历史：使用聊天历史来迭代之前的提示，避免重新开始。

10. 有目的地选择模型：根据需要选择不同模型，Gemini 适合精确任务，Claude 适合广泛应用。

11. 学习新技术栈：使用文档链接，让 Cursor 逐行解释错误和修复方法。

12. 让项目先索引完：让 Cursor 限制上下文范围以保持性能。

将 Cursor 代理视为强大的初级员工——如果你展示正确的方法，它可以走得很远，很快。
